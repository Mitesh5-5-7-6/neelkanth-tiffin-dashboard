# Prompt: Generate Production-Grade Invoice PDF System (Next.js App Router)

## Goal

Generate a complete, production-ready invoice PDF generation system for a Next.js App Router project using TypeScript, Tailwind CSS and Puppeteer. The output must be modular, well-typed, and match the provided invoice design (spacing, typography, rounded cards, gradients, shadows, A4 print layout). The PDF must be selectable text, use high-quality fonts, render print backgrounds (gradients) and produce a sharp QR code.

## High-level architecture (must follow)

- React invoice components (reusable) assembled into a print-optimized page
- Dedicated print page: `/invoice/[id]/print` (server-rendered, minimal client JS)
- Human-friendly viewer page: `/invoice/[id]` (optional interactive controls)
- Server API route: `/api/invoice/[id]` — generates A4 PDF via Puppeteer
- Helper: `lib/pdf/generatePdf.ts` to centralize Puppeteer options

## Strict constraints

- Use: Next.js App Router, TypeScript, Tailwind CSS, Puppeteer (headless Chromium)
- Do NOT use: jsPDF, html2canvas, pdf-lib, or canvas rendering
- PDF output: A4, `printBackground: true`, `preferCSSPageSize: true`, margins as specified
- Resulting PDF must be selectable text, compatible with Chrome and Acrobat
- Provide downloadable filename and proper response headers

## Files & folders to create (required)

- `app/invoice/[id]/page.tsx` — viewer page with "Download PDF" control
- `app/invoice/[id]/print/page.tsx` — print-optimized invoice layout (server component)
- `app/api/invoice/[id]/route.ts` — API endpoint returning `application/pdf`
- `components/invoice/*` — reusable components:
  - `InvoiceHeader.tsx`
  - `CustomerCard.tsx`
  - `StatsCards.tsx`
  - `InvoiceTable.tsx`
  - `PaymentDetails.tsx`
  - `AmountSummary.tsx`
  - `FooterSection.tsx`
  - `InvoiceDocument.tsx` (composes the above)
- `lib/pdf/generatePdf.ts` — Puppeteer wrapper (A4 + print options)
- `lib/mock/invoiceMock.ts` — mock invoice JSON generator for examples
- `lib/utils/qr.ts` — helper to generate inline QR (SVG/PNG data-URL)
- `types/invoice.type.ts` — TypeScript types for invoice data
- Update `app/globals.css` — add `@page` A4 rules and `@media print` styles

## Design & Layout Requirements (implement exactly)

- Header: left logo + company name/tagline; right invoice title, paid badge & metadata
- Summary cards: 4 cards (Total Days, Total Tiffins, Total Amount, Paid Amount) with soft gradients, rounded-xl and subtle shadows
- Table: columns Date/Day/Morning/Evening/Total/Rate/Amount; alternating row colors, fixed column alignment, print-safe borders
- Payment summary: right-side card with totals, subtotal, discounts, other charges and big TOTAL AMOUNT box
- Footer: company details / QR code / social icons / thank-you note

## Print CSS Requirements

- Add `@page { size: A4; margin: 10mm; }` and `body { -webkit-print-color-adjust: exact; color-adjust: exact; }`
- Avoid page-breaks inside the main invoice card. Use `page-break-inside: avoid` on main containers
- Use print-safe units (mm, px) where needed to guarantee consistent A4 rendering

## Puppeteer options (must use)

Use the following options in the Puppeteer helper (exposed or hard-coded):

await page.pdf({
format: 'A4',
printBackground: true,
preferCSSPageSize: true,
margin: { top: '0', right: '0', bottom: '0', left: '0' },
});

Notes: `printBackground: true` is required to preserve gradients/shadows in print output.

## API contract

- `GET /api/invoice/[id]` should return a PDF buffer with headers:
  - `Content-Type: application/pdf`
  - `Content-Disposition: attachment; filename="invoice-<id>.pdf"`
- Accept optional query params: `?download=1` (force download), `?format=a4` (future-proof)

## Ambiguities to clarify before generating code

- Where should generated PDFs be stored/archived (S3, local disk, DB)? Default: return as response and leave storage to a follow-up step.
- Should the API also support POST (send invoice data) vs only GET using server-side data? Default: implement GET for mock/demo, design to accept posted JSON in next iteration.

## Developer hints

- Prefer server components for print page to ensure static HTML with minimal client JS
- Use `-webkit-print-color-adjust: exact` for preserving print colors
- Use `page-break-inside: avoid` on main invoice card wrapper to keep it on single page
- In Puppeteer, use `waitUntil: 'networkidle0'` before `page.pdf()` to ensure fonts/QRs load
- Add `--no-sandbox` and `--disable-setuid-sandbox` to `puppeteer.launch()` for many Linux/container hosts
